## Task Management

1. **Plan First**: Write plan to tasks/ with checkable items. Name the markdown files with the name of the task
2. **Split tasks**: infrastructure changes should be isolated to only interconnected components - if the changes required are separate, split them into different tasks.
3. **Verify Plan**: Check in before starting implementation
4. **Track Progress**: Mark items complete as you go
5. **Explain Changes**: High-level summary at each step of why this
6. **Document Results**: Add review section to the markdown
7. **Capture Lessons**: Update `tasks/lessons.md after corrections or save to memory


## General workflow
1. Locate the proper place for the code - check modules, submodules
2. Follow the style of the code
3. Always run fmt and validate after changes
4. If there is no remote backend - run plans locally and validate output.
5. If there is a remote/cloud backend - use existing tools to validate changes.

## Subagents
Split work tasks with subagents for exploratory and research work - one task or topic per subagent.
